# Logist-
Программа для оптимизации логистического процесса. Строит маршруты между городами с помощью обхода графа в глубину.
Программа написана в 10 классе в качестве индивидуального проекта.
Основная функция - Route_s(). В ней строятся маршруты между двумя городами через промежуточные города на основе информации из файлов Towns.csv (соответствие городов, их координат, субъектов и численности населения) и Distances.csv - Расстояние между городами, высчитанное предварительно с помощью координат по формуле из сферической геометрии с коэффициентом 1,1 (коэфф. непрямолинейности дорог).
Кроме того, на входе программа спрашивает сведения об автопарке владельца, ключевой фактор (в рамках реализации) - расход топлива (л/100 км) и объём бака (л).
На выходе программа укажет цепочку городов маршрута, с указанием объёма дозаправки (в литрах).
Алгоритм:
1. Сеть городов, хранящихся в Towns.csv, можно представить в виде графа.
2. Из каждого города можно попасть в любой, если между ними есть дорога - информация о длине дорог хранится в таблице Distances.csv
3. Рекурсивно делаем шаг в город, в который достаточно объёма бака и есть дорога.
4. Высчитываем количество необходимого топлива, чтобы добраться до этого города.
- Это NP-трудная задача и при большой глубине (больше 4, если учитывать при построении все 1000+ городов в базе) программа работает очень долго. Во время участия в хактоне "КвантовыйМИР/Э/АТОМ" в треке "Оптимизация дорожного трафика в городе" в октябре 2025 года я придумал эвристический алгоритм: почему бы не разделить дороги по их трафику (чем больше трафик, тем она важнее). Таким образом, сначала выводить пользователя с наименее значимого узла на крупную дорогу, а затем от неё вести по просекам. Где именно сворачивать, можно посчитать с помощью метода координат.
